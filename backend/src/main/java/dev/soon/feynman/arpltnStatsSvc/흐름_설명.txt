데이터 흐름과 역할 분담
릴레이 경주처럼 각 계층은 자신의 역할만 수행하고 다음 계층에 결과를 전달합니다.

ArpltnStatsApiClient (API 레이어):

역할: 외부 API와 통신하는 데만 집중합니다.

흐름: API 엔드포인트에 요청을 보내고, 받은 JSON 응답을 ArpltnStatsApiResponse (원본 DTO) 객체로 변환하는 역할을 합니다. 응답을 받자마자 바로 상위 계층인 서비스로 넘겨줍니다.

ArpltnStatsService (서비스 레이어):

역할: 비즈니스 로직을 처리하는 핵심 계층입니다.

흐름: ArpltnStatsApiClient를 호출하여 외부 API 응답을 받습니다. 이 원본 데이터를 우리가 정의한 **ArpltnStatsResponse (표준화 DTO)**로 변환하고, 데이터베이스에 저장하는 등의 작업을 수행합니다.

Controller (프레젠테이션 레이어):

역할: 클라이언트(프론트엔드)의 요청을 받아 적절한 서비스를 호출하고 결과를 반환합니다.

흐름: 사용자 요청에 따라 ArpltnStatsService를 호출하고, 서비스가 반환한 표준화된 데이터를 클라이언트에 전달합니다.
